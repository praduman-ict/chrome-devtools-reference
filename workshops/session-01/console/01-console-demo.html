Console Panel Demo

Open DevTools Console with Ctrl+Shift+J or Cmd+Option+J, then click each demo in order.

1. Getting Started

After clicking, run window.demoState in the Console to inspect the page context.

No page state created yet.

2. Live Expressions

Create live expressions with the eye icon and try:

window.demoState.clicks
window.demoState.lastAction
document.querySelectorAll('.generated-log').length
document.activeElement

3. Console Methods

Use the Console level filters to show only info, warnings, or errors.

4. Styling Console Output

Uses %c placeholders and CSS strings.

5. Organizing Messages

Groups keep noisy debug output readable, and tables make arrays of objects easier to compare.

6. Filtering and Searching

Try these in the Console filter box:

url:01-console-demo.html
url:session-01-console-url-filter-demo.js
-url:extension

Use Ctrl+F or Cmd+F to search visible Console messages.

7. Understand Errors and Warnings

Hover a supported warning or error message and click Understand this warning or Understand this error.

Console Insights may require Chrome sign-in, English (US), AI Innovations enabled, and a supported region.

8. Console Utility Functions

Select one card in Elements, then try these commands in the Console:

$0
$('button')
$$('.utility-card')
copy(window.demoState)
getEventListeners(document.querySelector('#utility-click-target'))
monitorEvents(document.querySelector('#utility-click-target'), 'click')

9. Performance Profiling

Try profile('demo'), click the expensive function, then run profileEnd('demo').

10. Advanced Debugging

Run debug(processDemoData), click the button, then run undebug(processDemoData).
