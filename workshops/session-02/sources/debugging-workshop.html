7.4 Global DOM Removal Intercept

Problem: Break on → Node removal only works if you can right-click the element before it is removed. If a node disappears during page load, inside a framework teardown, or from a third-party script, it is gone before you can set the breakpoint.

Solution: Patch Element.prototype.remove to intercept every removal globally — no need to find the element first.

Intercept is OFF — removals will not be caught.
Intercept target — click "Remove an Element" to trigger removal

How it works (runs when you click Enable):

const originalRemove = Element.prototype.remove;
Element.prototype.remove = function () {
    console.trace('remove() called on', this);
    debugger;  // pauses here for every .remove() call
    return originalRemove.call(this);
};

Note

This same technique works directly in the Console or as a Sources Snippet on any page — not just this demo.

To intercept only a specific element instead of all elements:

const targetElement       = document.querySelector('.target');
const originalRemoveMethod = targetElement.remove.bind(targetElement);
targetElement.remove = function () { debugger; originalRemoveMethod(); };

To remove the global intercept from the Console:

Element.prototype.remove = originalRemove;

Or use Chrome's built-in Console helpers (no code needed):

debug(Element.prototype.remove);    // enable
undebug(Element.prototype.remove);  // disable
