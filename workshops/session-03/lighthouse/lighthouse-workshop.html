Lighthouse Workshop — Audit This Page

Instructions: Open DevTools → Lighthouse tab → select all categories → choose Desktop → click Analyze page load. Read the report section by section using the guide below.

1. Running the Audit

  1. Open DevTools (F12).
  2. Click the Lighthouse tab (may be hidden under ≫).
  3. Make sure all five categories are checked: Performance, Accessibility, Best Practices, SEO, PWA.
  4. Select Desktop for this workshop (fewer network-throttle variables).
  5. Click Analyze page load. Wait ~30–60 seconds.
  6. The report opens automatically. Do not close it — you will read through it below.

Run it twice and compare the Performance score — it varies slightly between runs due to CPU load and network timing. Take the lower of two runs as the realistic value.

3. Accessibility — Issues Planted on This Page

3.1 Low Contrast Text

⚠ Issue: the text below uses colour #aaaaaa on white — contrast ratio ~2.3:1, which fails WCAG AA (minimum 4.5:1).

This text is hard to read for people with low vision. Lighthouse will flag this under Accessibility → "Background and foreground colors do not have a sufficient contrast ratio."

Fix: change the colour to at least #767676 (4.5:1 on white) or darker.

3.2 Form Input Without a Label

⚠ Issue: the input below has no associated <label>. Screen readers cannot announce what the field is for.

Fix: add <label for="search-field">Search</label> and id="search-field" on the input.

3.3 Good Accessibility Patterns on This Page

  • <html lang="en"> — tells screen readers the page language.
  • Single <h1> at the top, followed by <h2> and <h3> — correct heading hierarchy.
  • All buttons have visible text labels.
  • Image above has a descriptive alt attribute.

4. SEO — Issues Planted on This Page

⚠ Issue: this page has no <meta name="description"> tag. Lighthouse will flag this under SEO → "Document does not have a meta description."

Fix: add inside <head>:
<meta name="description" content="A workshop page for practising Chrome DevTools Lighthouse audits.">

Good SEO Patterns on This Page

  • Page has a descriptive <title> tag.
  • <meta name="viewport"> is present — mobile-friendly.
  • Heading hierarchy is correct.
  • lang attribute is set on <html>.

6. Opportunities vs Diagnostics

The report splits findings into two groups:

Opportunities — directly improve the Performance score if fixed.

Each opportunity shows an estimated saving in milliseconds. Fix the highest-saving one first.

Diagnostics — additional context that explains the score but does not directly map to a millisecond saving.

Examples: DOM size, main-thread work, third-party code usage.

8. Mobile vs Desktop Mode

Run the audit again with Mobile selected. Lighthouse applies simulated mobile throttling (slower CPU, slower network). The Performance score will likely drop significantly compared to Desktop.

Why the difference matters:

Most web traffic is mobile. A desktop score of 95 might correspond to a mobile score of 60. Always run both and treat the Mobile score as the primary health indicator.
