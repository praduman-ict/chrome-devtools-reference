Overrides - Workshop

Setup first: Sources panel → Overrides tab (in the left sidebar) → + Select folder for overrides → choose any empty folder on your machine → click Allow.

1. What Overrides Do

Without Overrides, any edit you make in the Sources panel is lost on the next reload. With Overrides enabled and a folder selected, edits are saved to your local folder and the browser serves your local copy instead of the remote file — even after a reload.

When to use Overrides:

  • Test a CSS or JS fix on a live site before you have access to the source code.
  • Mock an API response when the backend endpoint is not ready yet.
  • Toggle a feature flag without deploying.
  • Reproduce a bug with a patched version of the file.

4. Mocking an API Response

The button below fetches /api/workshop-products — a route that does not exist on this server. Use Overrides to create a local file that intercepts the request and returns mock JSON.

Products will appear here.

How to mock the API:

  1. In your override folder, mirror the URL path: create <override-folder>/<hostname>/api/workshop-products (no extension).
  2. Paste this JSON into the file: 
    [
  { "id": 1, "name": "Notebook", "price": 120 },
  { "id": 2, "name": "Pen Set",  "price": 80  }
]
  3. Reload and click Load Products — the browser serves your local file instead of hitting the server.

Why this matters: you can develop and test UI behavior against realistic API data before the backend endpoint exists.

6. Verifying and Removing Overrides

How to tell an override is active

  • The file in Sources shows a purple dot icon next to its name.
  • The Overrides tab lists all currently active override files.

How to remove a specific override

  1. Delete the file from your override folder on disk.
  2. Reload — the browser fetches the original remote file again.

How to disable all overrides at once

  1. Sources → Overrides tab → uncheck Enable Local Overrides.
  2. All overrides are suspended (files still exist in the folder).
  3. Re-check to reactivate them.

Warning: Overrides stay active across browser restarts. Always check the Overrides tab before debugging a "strange" page behavior — a forgotten override might be the cause.
